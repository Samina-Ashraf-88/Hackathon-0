# SKILL: Daily Briefing

## Description

Generates a concise, actionable daily briefing by scanning all active vault folders. Outputs a human-readable report, updates `Dashboard.md`, and saves the briefing to `/Briefings/`. Transforms the AI Employee from reactive to proactive — this is the "Monday Morning CEO Briefing" feature.

---

## When to Use

- Every morning (scheduled via Windows Task Scheduler or cron)
- User says: *"Give me a briefing"*, *"What needs attention?"*, *"Status update"*
- After a batch of files has been processed
- Weekly: for the full Business Audit version (Gold tier extension)

---

## Inputs Required

| Source | Path | What to Scan |
|--------|------|-------------|
| Pending files | `Needs_Action/` | Count + list unprocessed items |
| Approval queue | `Pending_Approval/` | Items requiring human decision |
| Completed today | `Done/` | Files completed in last 24h |
| Active plans | `Plans/` | Plans with status ≠ complete |
| Business goals | `Dashboard.md` | Current metrics snapshot |

---

## Step-by-Step Procedure

**Step 1 — Scan all folders**
```
Count files in: Needs_Action/, Pending_Approval/, Done/, Plans/
For Pending_Approval: read each file's action, amount, deadline fields
For Plans: read each file's status and next unchecked step
```

**Step 2 — Calculate metrics**
```
pending_count     = count of FILE_*.md in Needs_Action/
approval_count    = count of files in Pending_Approval/
completed_today   = count of files in Done/ modified today
active_plans      = count of plans with status != complete
```

**Step 3 — Draft the briefing**
```
Follow the Briefing Output Schema below.
Be concise. Lead with what the HUMAN needs to do.
```

**Step 4 — Save briefing**
```
Save to: Briefings/BRIEFING_{YYYY-MM-DD}.md
If briefing for same date exists, append with timestamp suffix.
```

**Step 5 — Update Dashboard.md**
```
Update these sections:
  - Needs Action Queue (count + list)
  - Pending Approvals (count + list)
  - Recent Activity (add briefing generation entry)
  - System Status (mark Claude Reasoning as ✅ Active)
```

**Step 6 — Log the action**
```
Write to Logs/{YYYY-MM-DD}.json:
{"timestamp":"...","action_type":"daily_briefing_generated","actor":"claude_code","result":"success"}
```

## Scheduling (Windows Task Scheduler)

To run the briefing every morning at 8:00 AM:

```powershell
# Create scheduled task (run once in PowerShell as Admin)
$action = New-ScheduledTaskAction -Execute "claude" -Argument '--print "Read Skills/SKILL_Daily_Briefing.md and Company_Handbook.md. Generate today briefing. Save to Briefings/. Update Dashboard.md."' -WorkingDirectory "E:\hackathon-0"
$trigger = New-ScheduledTaskTrigger -Daily -At "08:00AM"
$settings = New-ScheduledTaskSettingsSet -StartWhenAvailable
Register-ScheduledTask -TaskName "AI_Employee_Daily_Briefing" -Action $action -Trigger $trigger -Settings $settings
```

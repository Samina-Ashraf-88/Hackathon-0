# SKILL: Generate Plan

## Description

This skill creates a structured `Plan.md` file with a clear objective, checkbox steps, approval flags, and completion criteria. **Every multi-step Claude action must begin with a plan.** Plans are the audit trail of intent before execution.

---

## When to Use

- Processing any dropped file that requires more than one action
- Responding to a task that touches external systems
- User says: *"Create a plan for..."* or *"Plan how to..."*
- Before any action flagged as requiring approval in Company Handbook Section 3
- Called by `SKILL_Process_File_Drop` as Step 4

---

| Input | Source | Required |

## Plan File Schema

Every plan file must follow this exact structure:

```markdown
---
plan_id: PLAN_{task_slug}_{YYYYMMDD_HHMMSS}
created: {ISO 8601 timestamp}
triggered_by: {path/to/triggering/file or "user prompt"}
status: pending
requires_approval: true | false
approval_items: []
skill_used: SKILL_Generate_Plan v1.0
---

# Plan: {Human-readable Task Title}

## Objective
{One clear sentence: what this plan achieves when complete.}

## Context
- **Triggered by:** {source file or event}
- **File type:** {if applicable}
- **Handbook sections:** {e.g., "Section 3 (Action Boundaries), Section 2 (Financial Rules)"}
- **Deadline:** {if any, else "None"}
- **Estimated steps:** {count}

## Steps

- [ ] Step 1: {Clear, specific action — include target file/folder}
- [ ] Step 2: {Clear, specific action}
- [ ] Step 3: {Clear, specific action}
- [ ] Step N: Move source metadata to /Done/ (final step — always last)

## Approval Required

{Yes / No}

{If Yes — list each item requiring approval:}
- Item: {description} | Reason: {handbook rule reference}

## Completion Criteria

This plan is complete when ALL of the following are checked:
- [ ] All steps above are marked complete
- [ ] Source file metadata moved to /Done/ OR approval file in /Pending_Approval/
- [ ] Dashboard.md updated under "Recent Activity"
- [ ] Log entry written to /Logs/{YYYY-MM-DD}.json

## Notes

{Observations, warnings, handbook rule citations, or recommendations.}
{Flag anything unusual for human review.}

---
*Generated by AI Employee | Skill: SKILL_Generate_Plan v1.0 | {timestamp}*
```

## Step-by-Step Procedure

**Step 1 — Gather context**
```
Read: Company_Handbook.md (Sections 3, 5, 7)
Read: Dashboard.md (current state)
Read: triggering file / task description
```

**Step 2 — Define objective**
```
Write ONE sentence: "This plan achieves X by doing Y."
```

**Step 3 — Decompose into steps**
```
Break the task into 3–8 atomic steps.
Each step must be:
  - Actionable (starts with a verb: Read, Write, Create, Move, Draft, Log)
  - Specific (includes exact file path or target)
  - Bounded (one action per step)
The FINAL step is ALWAYS: "Move [source] to /Done/"
```

**Step 4 — Check for approval requirements**
```
For each step, check Company_Handbook.md Section 3.
If ANY step requires approval → set requires_approval: true
  and add that step to approval_items list.
```

**Step 5 — Set status and save**
```
status: "pending" (not yet started)
Save to: Plans/PLAN_{task_slug}_{timestamp}.md
```

**Step 6 — Return the plan path to caller**
```
Output: "Plan created at: Plans/PLAN_{name}_{ts}.md"
```

```
You are an AI Employee. Create a Plan.md file for the following task.

Task description: {TASK_DESCRIPTION}
Triggered by file: {SOURCE_FILE_PATH}
File type: {FILE_TYPE}

Instructions:
1. Read Company_Handbook.md (Sections 3, 5, 6, 7).
2. Create a plan following the schema in Skills/SKILL_Generate_Plan.md.
3. Break the task into 3–8 atomic checkbox steps.
4. Set requires_approval: true if ANY step touches: email sending, payments, external APIs, or deletion.
5. The final step must always be: Move metadata to /Done/.
6. Save the plan to: Plans/PLAN_{task_slug}_{YYYYMMDD_HHMMSS}.md
7. Return the path of the created plan.
```

## Naming Convention

```

Examples:
  PLAN_report_pdf_20260223_143022.md
  PLAN_client_invoice_20260223_091500.md
  PLAN_subscription_audit_20260223_080000.md
```

---

## Status Values

| Status | Meaning |
|--------|---------|
| `pending` | Plan created, not started |
| `in_progress` | Claude is executing steps |
| `awaiting_approval` | Blocked on human approval |
| `complete` | All steps done, moved to Done |
| `cancelled` | Human rejected or cancelled |
